# 项目档案

## 基本信息

- **项目一句话**：一个 Node CLI，用于把 CSV 导出为 Markdown 报告
- **项目类型**：`CLI`
- **主分支 / 发布方式**：`main` / 发布 npm 包
- **主工作区路径约定**：单一主工作区，不使用额外 worktree
- **仓库布局**：`单仓`

## 运行时与语言

- **语言与最低版本**：Node 20+
- **包管理 / 锁文件**：`pnpm`
- **AI 引入新依赖**：`须用户每次确认`

## 源码与产物

- **主要源码路径**：`src/**`
- **生成物 / 禁止手改**：`dist/**`
- **构建命令**：`pnpm build`
- **类型检查 / 静态检查**：`pnpm typecheck`
- **测试命令**：`pnpm test`
- **CI**：`.github/workflows/ci.yml`；本地最小集合：`pnpm typecheck && pnpm test`

## 部署或同步

无单独部署步骤；`commit` 后由 CI 发预发布包。

## 严审触发

默认规则之外，本项目追加：

- CLI 参数格式或输出格式变更
- 文件系统写入路径变更

## 按项目类型的默认关注项

### CLI / 工具

- [x] 支持的最低运行时版本：Node 20
- [x] 配置加载顺序：CLI 参数 > 环境变量 > 配置文件
- [x] 输出格式：TTY 彩色，管道时纯文本
- [x] 退出码约定：`0` 成功，`1` 用户输入错误，`2` 运行时失败
- [ ] 是否支持插件/扩展机制：否

## 设计文档位置

- **设计/方案笔记目录**：`docs/design/`

## 删除备份

- **备份根目录**：仓库根 `.ai-deletion-backups/`
- **保留天数**：30
- **清理责任方**：`human`

## 其他约束

- 不得把真实客户 CSV 示例提交到仓库
- `.gitignore` 应包含 `.ai-deletion-backups/`
