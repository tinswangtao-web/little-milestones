# Cursor Review 输出模板（固定版）

用于：Cursor 对 Codex 改动做审查  
目标：让用户（非技术）也能一眼判断“能不能发”

## Review 结论

- 结论：`可发布 / 需修复 / 建议回滚`
- 风险等级：`低 / 中 / 高`
- 是否阻塞提交：`是 / 否`

> 任何会导致错误行为、数据风险、明显回归的问题

- [P0-1] 标题：
  - 现象：
  - 影响范围：
  - 复现步骤：
  - 建议修法（最小改动）：
  - 是否已复测通过：`是/否`

- [P0-2] 标题：
  - 现象：
  - 影响范围：
  - 复现步骤：
  - 建议修法（最小改动）：
  - 是否已复测通过：`是/否`

## 4) 变更核对（给用户看的）

- 本轮目标是否达成：`是/否`
- 是否出现新回归：`有/无`
- 是否影响历史数据：`是/否`
- 是否需要迁移/清理：`是/否`

## 5) 验证记录（Codex已做）

- Build：`通过/失败`
- Typecheck/Lint：`通过/失败`
- 手工验证场景：
  - [ ] 场景A
  - [ ] 场景B
  - [ ] 场景C

## 6) 给用户的下一步（非技术）

- 你现在只要做：
  1. ...
  2. ...
- 看到以下结果就算通过：
  - ...
- 若不符合，请回我这句话：
  - “第X步不符合，现象是 ...（可附截图）”

## 7) 提交建议

- 建议现在提交：`是/否`
- 建议先同步Vault再测：`是/否`
- 建议commit message（如需）：
  - `[codex] ...`
